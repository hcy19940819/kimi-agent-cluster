## 三、删除的规则清单

以下规则在旧架构中存在（明文或隐含），在新架构中被删除：

| 删除的规则 | 删除原因 | 替代方案 |
|-----------|---------|---------|
| "复杂正则用 ReadFile" | 过细的操作指南，不是约束 | 融入AI默认能力 |
| "跨文件搜索用 Grep" | 过细的操作指南，不是约束 | 融入AI默认能力 |
| "禁止标记 completed (assumed)" | 过细，融入 R10 的标记规范 | R10 涵盖 |
| "GPU并行度限制" | 属于配置，不是规则 | CONTEXT.md #Constraints |
| "存储分层顺序" | 属于配置，不是规则 | CONTEXT.md #Constraints |
| "分层开发顺序" | 属于项目计划，不是约束 | 外部工具 |
| "6个安全检查脚本清单" | 脚本化后不需记录清单 | verify.sh |
| "沟通风格" | 属于AI行为设定 | 删除（默认行为） |
| "开发策略" | 属于项目计划 | 外部工具 |

**删除原则**：
- 过细的操作指南 → 删除（AI应该自己会）
- 属于配置/计划的 → 移到 CONTEXT.md 或外部工具
- 已被脚本覆盖的 → 删除（脚本即是规则）
