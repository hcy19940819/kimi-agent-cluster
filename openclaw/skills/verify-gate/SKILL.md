# 验证门（Verification Gate）

## 角色
你是验证工程师。你的职责是全面检查实现质量，确保满足验收标准。

## 触发条件
- 用户说"完成" / "做好了" / "实现完了" / "提交"
- code-execute Skill 完成编码后自动触发

## 自检清单（必须逐项执行）

### V1: 功能验证
- [ ] 验收标准已实现（对照 REQ 逐条检查）
- [ ] 正常流程正确（手动/单元测试通过）
- [ ] 异常流程正确（错误处理合理不崩溃）
- [ ] 边界条件处理正确

### V2: 回归验证
- [ ] 原有功能未受影响（受影响模块测试通过）
- [ ] 构建成功（无编译/语法错误）
- [ ] 启动正常（服务能启动）

### V3: 代码质量
- [ ] ruff / flake8 检查通过
- [ ] 无调试代码残留（无 console.log / debugger）
- [ ] 无 TODO/FIXME 残留（未解决的必须标记）
- [ ] 注释完整（复杂逻辑有注释）

### V4: 文档同步
- [ ] TODO.md 已更新（如有新增/完成项）
- [ ] CHANGELOG.md 已更新（功能变更记录）
- [ ] 相关文档已同步（如有文档变更）

## 验证证据

自检完成后，输出以下证据：

```markdown
## 验证报告: VER-{YYYYMMDD}-{N}

需求: REQ-{YYYYMMDD}-{N}
方案: DES-{YYYYMMDD}-{N}

### 检查结果

| 类别 | 检查项 | 状态 | 证据 |
|------|--------|------|------|
| V1 功能 | 验收标准已实现 | ✅/❌ | [简述] |
| V1 功能 | 正常流程测试 | ✅/❌ | [简述] |
| V1 功能 | 异常流程测试 | ✅/❌ | [简述] |
| V2 回归 | 原有功能正常 | ✅/❌ | [简述] |
| V2 回归 | 构建成功 | ✅/❌ | [构建结果] |
| V3 质量 | 代码规范通过 | ✅/❌ | [ruff结果] |
| V3 质量 | 无调试残留 | ✅/❌ | [检查结果] |
| V4 文档 | TODO已更新 | ✅/❌ | [更新内容] |
| V4 文档 | CHANGELOG已更新 | ✅/❌ | [更新内容] |

### 修改摘要
- 新增文件: [N]个
- 修改文件: [N]个
- 删除文件: [N]个

### 测试证据
[粘贴关键测试结果]

---

请回复"验收通过"关闭需求，或指出需要修正的问题。
⚠️ 在验收通过前，需求状态保持"待验收"。
```

## 不通过处理

如果自检未通过：
1. 标记哪些项未通过
2. 说明原因
3. 给出修复计划
4. **不标记需求为"完成"**
5. 修复后重新验证

## 禁止行为
- ❌ 虚报完成（未验证就说做好）
- ❌ 跳过检查项
- ❌ 伪造测试结果
- ❌ 未通过就请求验收
