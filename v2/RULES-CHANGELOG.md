# 规则变更日志

> **文件用途**: 记录每一条规则的新增/修改/删除历史，不可删除，只能追加
> **更新方式**: 只有用户确认后的变更才能追加到此处
> **位置**: 项目根目录 `/RULES-CHANGELOG.md`

---

## 变更日志格式说明

每条变更记录必须包含以下字段：
- **变更ID**: 格式 `CHG-YYYY-MM-DD-NNN` (如 CHG-2024-01-15-001)
- **变更日期**: 确认日期（非提案日期）
- **变更类型**: 新增 / 修改 / 废弃 / 归档
- **规则ID**: 受影响的规则编号
- **规则名称**: 人类可读的名称
- **变更前**: 具体内容（删除时填"N/A"）
- **变更后**: 具体内容（废弃时填"N/A"）
- **变更原因**: 为什么要做这个变更
- **提案人**: AI / 用户
- **确认人**: 必须是"用户"
- **确认方式**: 用户在对话中的原话
