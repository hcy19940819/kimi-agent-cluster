# 规则变更提案模板

> **文件用途**: AI拟变更规则时，必须按此格式提交给用户审批
> **使用方式**: AI将此模板填充后展示给用户，等待用户明确回复
> **禁止**: 未经用户明确回复"同意/批准/可以"等肯定词汇前，AI不得执行变更

---

## 提案单

```
═══════════════════════════════════════════════════
              规 则 变 更 提 案
═══════════════════════════════════════════════════

【提案编号】 PROP-YYYY-MM-DD-NNN
【提案日期】 YYYY-MM-DD HH:MM
【提案人】   AI (实例ID)
【紧急程度】 □ 紧急(阻塞开发) / □ 普通(建议采纳) / □ 低(观察记录)

───────────────────────────────────────────────────
一、变更摘要
───────────────────────────────────────────────────

我想申请 [新增/修改/废弃] 以下规则：

  规则层次: [元规则 / 执行规则 / 场景规则]
  规则ID:   [如 E4 / S2 等，由AI按层次自动编号]
  规则名称: [一句话概括]
  当前状态: [如果是修改/废弃]

───────────────────────────────────────────────────
二、变更详情
───────────────────────────────────────────────────

【变更类型】新增 / 修改 / 废弃

【规则全文】（如果是新增/修改，写出完整规则内容）

  > 此处写规则的具体内容

【变更前】（如果是修改/废弃）

  > 原规则内容或"N/A"

【变更后】（如果是修改/废弃）

  > 新规则内容或"N/A"

───────────────────────────────────────────────────
三、变更理由
───────────────────────────────────────────────────

【触发原因】
  这次变更是因为：
  □ P0/P1事故后的经验教训
  □ 开发效率问题
  □ 代码质量问题
  □ 用户主动提出
  □ 其他: _______

【为什么需要变成规则】
  （AI说明为什么这个经验教训需要固化成规则，
   而不是放在MEMORY.md作为参考）

【如果不加这条规则会怎样】
  （AI分析不添加的潜在风险）

【是否有替代方案】
  （AI说明是否有更轻量的方案，如检查清单、模板等）

───────────────────────────────────────────────────
四、影响分析
───────────────────────────────────────────────────

【对现有规则的影响】
  □ 与现有规则无冲突
  □ 替代/废弃现有规则: [规则ID]
  □ 需要修改现有规则以配合
  □ 可能与某规则冲突，需裁决

【对开发流程的影响】
  □ 无影响（纯记录性规则）
  □ 轻微影响（增加一个检查步骤）
  □ 中等影响（改变现有流程）
  □ 重大影响（颠覆性变更）

【规则数量影响】
  变更前总数: XX / 20
  变更后总数: XX / 20
  是否触发膨胀警告: □ 是 / □ 否

───────────────────────────────────────────────────
五、实施计划
───────────────────────────────────────────────────

【生效方式】
  □ 立即生效（用户确认后立即写入）
  □ 试运行7天（7天后自动转正，除非用户叫停）
  □ 指定日期生效: YYYY-MM-DD

【需要更新的文件】
  □ RULES-GOVERNANCE.md (规则看板)
  □ RULES-CHANGELOG.md (变更日志)
  □ AGENTS.md (如元规则变更)
  □ MEMORY.md (如经验教训同步)
  □ 其他: _______

【实施步骤】
  1. 用户确认
  2. AI更新对应规则文件
  3. AI更新 RULES-GOVERNANCE.md 看板
  4. AI追加 RULES-CHANGELOG.md 日志
  5. 7天后（如试运行）AI自动转正并更新状态

═══════════════════════════════════════════════════

【用户确认区 - 请用户填写】

  我已阅读以上提案：

  □ 批准 — 同意实施，按上述计划执行
  □ 试运行 — 同意试运行7天，7天后自动转正
  □ 否决 — 不同意，理由: ________________
  □ 修改 — 需要调整: ____________________
  □ 暂缓 — 先不处理，原因: ______________

  我的决定: ________________
  补充说明: ________________

═══════════════════════════════════════════════════
```

## 用户确认的快速模式

对于小额变更（单条执行规则/场景规则），可以使用简化版：

```
═══════════════════════════════════════════════════
【快速提案】我想 [新增/修改/废弃] 一条规则：

层次: [元/执行/场景] | ID: [编号] | 名称: [名称]

规则内容: [一句话]
理由: [为什么]
变更后总数: X/20 [是否超标]

请确认: [同意/否决/需要修改] ?
═══════════════════════════════════════════════════
```

## 禁止事项（AI必须遵守）

AI在提案过程中绝对禁止：

1. ❌ **禁止先斩后奏**: 不得先修改规则文件，再"告知"用户
2. ❌ **禁止默认同意**: 用户沉默 ≠ 同意，必须获得明确肯定答复
3. ❌ **禁止拆分规避**: 不得将一条规则拆成多条以规避数量上限
4. ❌ **禁止紧急绕过**: 不得以"紧急"为借口绕过确认流程
5. ❌ **禁止间接植入**: 不得在普通文档中偷偷写入规则性语言（如"必须""禁止""每次都要"）
6. ❌ **禁止强制升级**: 不得将经验教训自动升级为"强制执行规则"

## AI在收到用户确认后的执行清单

```
□ 1. 在RULES-GOVERNANCE.md中更新规则条目
□ 2. 在RULES-GOVERNANCE.md中更新统计数字
□ 3. 在RULES-CHANGELOG.md中追加变更记录
□ 4. 在对应规则文件中写入/修改规则内容
□ 5. 在执行清单中标注"已完成"并引用确认原话
□ 6. 如果试运行，设置7天提醒标记
```
